# Generated Wall Tile Assets

This directory is generated by the offline wall tile pipeline. Do not hand-edit generated PNG or JSON outputs unless
you are intentionally testing a temporary local variant.

## Source

Canonical input source sheet:

```bash
custodian/content/tiles/walls/source/procgen_wall_modules_source.png
```

The original development intake file may live under `custodian/content/dev/in_progress/`; copy the reviewed source to
the canonical path above before regenerating.

## Generate Parts And Atlas

```bash
python3 tools/tiles/extract_wall_parts.py \
  --input custodian/content/tiles/walls/source/procgen_wall_modules_source.png \
  --out-dir custodian/assets/tiles/walls/generated/procgen_wall_source_parts \
  --atlas-out custodian/assets/tiles/walls/generated/procgen_wall_source_atlas.png \
  --json-out custodian/assets/tiles/walls/generated/procgen_wall_source_parts.json \
  --min-area 16
```

## Generate Composed Variants

```bash
python3 tools/tiles/compose_wall_variants.py \
  --parts-json custodian/assets/tiles/walls/generated/procgen_wall_source_parts.json \
  --atlas custodian/assets/tiles/walls/generated/procgen_wall_source_atlas.png \
  --out custodian/assets/tiles/walls/generated/procgen_wall_composed_variants.png \
  --count 32 \
  --seed 1234
```

## Generate Procgen Runtime Atlas

Passage strips live here:

```bash
custodian/content/tiles/walls/source/wall_passages/
```

They should be `32px` tall horizontal strips. The bridge slices them into consecutive `32x32` runtime cells and routes
them into passage/hole buckets for procgen wall openings.

```bash
python3 tools/tiles/build_procgen_wall_atlas.py \
  --parts-json custodian/assets/tiles/walls/generated/procgen_wall_source_parts.json \
  --atlas custodian/assets/tiles/walls/generated/procgen_wall_source_atlas.png \
  --semantics tools/tiles/procgen_wall_semantics.json \
  --passage-dir custodian/content/tiles/walls/source/wall_passages \
  --out-image custodian/content/tiles/walls/generated/procgen_wall_tiles_32.png \
  --out-json custodian/content/tiles/walls/generated/procgen_wall_tiles_32.mapping.json \
  --tile-size 32
```

## Manual Rectangles

If a future source sheet does not have useful transparency, provide explicit rectangles:

```bash
python3 tools/tiles/extract_wall_parts.py \
  --input custodian/content/tiles/walls/source/procgen_wall_modules_source.png \
  --rects path/to/wall_source_rects.json \
  --out-dir custodian/assets/tiles/walls/generated/procgen_wall_source_parts \
  --atlas-out custodian/assets/tiles/walls/generated/procgen_wall_source_atlas.png \
  --json-out custodian/assets/tiles/walls/generated/procgen_wall_source_parts.json
```

Rectangle config format:

```json
{
  "tile_size": 32,
  "parts": [
    {
      "id": "long_wall_01",
      "rect": [32, 48, 256, 96],
      "kind": "long_straight",
      "tags": ["stone"],
      "anchor": "bottom_left"
    }
  ]
}
```

## Godot Import Notes

Import generated PNGs as pixel art:

- filter off
- mipmaps off
- lossless compression
- no runtime scaling

Use the generated atlas through the normal Godot `TileSet` / atlas workflow. This is an offline content pipeline, not
runtime image generation.
